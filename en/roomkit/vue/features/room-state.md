## Feature Introduction

[useRoomState](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#module-RoomState) is the core Hook responsible for room management in the Atomicx solution. It encapsulates complex underlying signaling processes, and developers can drive room state through simple reactive data and method calls.
- **Create & Join:** Create and join a room through [createAndJoinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#createAndJoinRoom) method, or join an existing room through [joinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#joinRoom) method.

- **Real-time State Sync:** The `useRoomState` Hook internally maintains [currentRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#currentRoom) reactive data, automatically synchronizing room name, mode, member count and other information.

- **Role-based Exit Mechanism:** Provides clear operational boundaries for different roles. Regular members can use [leaveRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#leaveRoom) to exit; while the room owner (host) has [endRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#endRoom) permission to end the room and destroy room resources with one click.


## **Prerequisites**

User has completed login authentication through [useLoginState](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#module-LoginState), please refer to [Integration Overview](https://write.woa.com/document/197379797539790848).

## Implementing Room Management

### Step 1: Import useRoomState
``` typescript
import { useRoomState } from 'tuikit-atomicx-vue3/room';

const { 
  createAndJoinRoom, 
  joinRoom, 
  leaveRoom, 
  endRoom,
  currentRoom 
} = useRoomState();
```

### Step 2: Create Room

Create room (room owner): Use [createAndJoinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#createAndJoinRoom), after successful call the user will automatically obtain room owner permissions.

> **Note:**
> 

> The roomId specified when creating a room is recommended to be generated and issued by the business backend to ensure global uniqueness and stability of the room ID.
> 

- **Example 1: User creates room**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { createAndJoinRoom } = useRoomState();
   
   await createAndJoinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
   })
   ```
- **Example 2: User creates room and specifies room name**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { createAndJoinRoom } = useRoomState();
   
   await createAndJoinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
     options: {
       roomName: 'My Temporary Room'
     }
   })
   ```
- **Example 3: User creates password-protected room**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { createAndJoinRoom } = useRoomState();
   
   await createAndJoinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
     options: {
       roomName: 'My Temporary Room',
       password: '123456',
     }
   })
   ```   

   > **Note:**
   > 

   > Creator automatically joins the room when calling [createAndJoinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#createAndJoinRoom), no password required.
   > 

- **Example 4: User creates room and configures member management settings**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { createAndJoinRoom } = useRoomState();
   
   await createAndJoinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
     options: {
       roomName: 'My Temporary Room',
       password: '123456',
       isAllMicrophoneDisabled: true,   // Enable mute all
       isAllCameraDisabled: true,       // Enable disable all videos
       isAllScreenShareDisabled: true,  // Enable disable all screen sharing
       isAllMessageDisabled: true,      // Enable disable all in-meeting messages
     }
   })
   ```   

   > **Note:**
   > 

   > Member management configuration only applies to regular members, room owner and administrators are not restricted. To modify member management configuration after room creation or manage individual member permissions, please refer to [Member Management](https://write.woa.com/document/197380064613830656).
   > 


### Step 3: User joins existing room

Logged-in users can call [joinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#joinRoom) to join an existing room. Atomicx only supports users joining one room at a time.
- **Example 1: User joins room**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { joinRoom } = useRoomState();
   
   await joinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
   })
   ```
- **Example 2: User joins password-protected room**

   ``` typescript
   import { useRoomState } from 'tuikit-atomicx-vue3/room';
   const { joinRoom } = useRoomState();
   
   await joinRoom({
     roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
     password: '123456',
   })
   ```

### Step 4: Update room information

Room owner can use [updateRoomInfo](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#updateRoomInfo) to update room name and password after creating a room.
``` typescript
import { useRoomState } from 'tuikit-atomicx-vue3/room';
const { updateRoomInfo } = useRoomState();

await updateRoomInfo({
  roomId: 'YOUR_ROOM_ID',    // Specify room ID, generated by business backend
  options: {
     roomName: 'New Room Name',
     password: '234567',
  },
})
```

> **Note:**
> 

> Room owner must be in the room to call [updateRoomInfo](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#updateRoomInfo). If room owner has left the room, they need to rejoin before updating room information.
> 


### Step 5: Leave room

Both room owner and regular users can call [leaveRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#leaveRoom) interface to leave the room. Prerequisite for using this interface is that the user has joined the room through [createAndJoinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#createAndJoinRoom) or [joinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#joinRoom) interface.
``` typescript
import { useRoomState } from 'tuikit-atomicx-vue3/room';
const { leaveRoom } = useRoomState();

await leaveRoom();
```

### Step 6: Destroy room

Only room owner can call [endRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#endRoom) interface to destroy the room. Prerequisite for using this interface is that the user has created a room through [createAndJoinRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#createAndJoinRoom) or has obtained room owner status.
``` typescript
import { useRoomState } from 'tuikit-atomicx-vue3/room';
const { endRoom } = useRoomState();

await endRoom();
```

> **Note:**
> 
> - If room owner leaves the room without calling [endRoom](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#endRoom), the room will continue to exist until automatic reclamation conditions are met (6 consecutive hours without users).
> - Room owner can transfer owner status to other users through [transferOwner](https://web.sdk.qcloud.com/trtc/live/web/doc/zh/index.html#transferOwner) interface.


### Step 7: Handle abnormal exit

#### Handle room destroyed event

Regular users can handle room destruction business interaction by listening to `RoomEvent.onRoomEnded` event.
``` typescript
import { useRoomState, RoomEvent } from 'tuikit-atomicx-vue3/room';
import { TUIMessageBox } from '@tencentcloud/uikit-base-component-vue3';
const { subscribeEvent, unsubscribeEvent } = useRoomState();

function onRoomEnded() {
  TUIMessageBox.alert({
    title: 'Notice',
    content: 'Room owner has ended the room',
  });
}

// Subscribe to RoomEvent when component loads
subscribeEvent(RoomEvent.onRoomEnded, onRoomEnded);
// Unsubscribe from RoomEvent when component is destroyed
unsubscribeEvent(RoomEvent.onRoomEnded, onRoomEnded);
```

#### Handle user kicked from room event

Regular users can handle being kicked from room business interaction by listening to `RoomParticipantEvent.onKickedFromRoom` event.
``` typescript
import { useRoomParticipantState, RoomParticipantEvent, KickedOutOfRoomReason } from 'tuikit-atomicx-vue3/room';
import { TUIMessageBox } from '@tencentcloud/uikit-base-component-vue3';

const { subscribeEvent, unsubscribeEvent } = useRoomParticipantState();

function onKickedFromRoom({ reason }: { reason: KickedOutOfRoomReason; message: string }) {
    let notice = '';
    switch (reason) {
      case KickedOutOfRoomReason.KickedByAdmin:
        notice = 'You have been removed from the room by the host';
        break;
      case KickedOutOfRoomReason.ReplacedByAnotherDevice:
        notice = 'Your account has been logged in on another device';
        break;
      case KickedOutOfRoomReason.KickedByServer:
        notice = 'You have been removed from the room by the server';
        break;
      case KickedOutOfRoomReason.ConnectionTimeout:
        notice = 'Network connection timeout, exiting room';
        break;
      case KickedOutOfRoomReason.InvalidStatusOnReconnect:
        notice = 'Room has been destroyed or you were removed during offline period';
        break;
      case KickedOutOfRoomReason.RoomLimitExceeded:
        notice = 'Room capacity reached, unable to join';
        break;
      default:
        notice = 'You have been removed from the room';
        break;
    }
    TUIMessageBox.alert({
      title: 'Notice',
      content: notice,
    });
  }

// Subscribe to RoomParticipantEvent when component loads
subscribeEvent(RoomParticipantEvent.onKickedFromRoom, onKickedFromRoom);
// Unsubscribe from RoomParticipantEvent when component is destroyed
unsubscribeEvent(RoomParticipantEvent.onKickedFromRoom, onKickedFromRoom);
```

## Advanced Implementation

The current document introduces quickly creating and immediately joining a room, suitable for instant multi-party audio/video communication scenarios.

However, in real business scenarios, we often need to plan ahead: set room topics, schedule specific start and end times, and invite participants. Participants can receive reminders before the multi-party audio/video room starts and join on time.

Atomicx provides complete scheduled room capabilities, including:
- **Schedule Room:** Set multi-party room time and participant list in advance.

- **Schedule List**: View and manage all upcoming multi-party rooms under your name.

- **Meeting Reminders:** Support automatic reminder logic before start.


   If you need to implement more complex business multi-party audio/video room management logic as described above, please refer to: [Schedule Room](https://write.woa.com/document/197380092150673408).


## FAQ

### If there are no users in the room but the room has not been destroyed, will the room exist forever?

When there are no users in the room for 6 consecutive hours, the backend will attempt to reclaim the room.

### If there are no users in the room but the room has not been destroyed, will there be any charges?

Rooms without users but not yet destroyed only occupy package group count, do not consume minutes and do not generate additional charges.

### Can a user join multiple rooms simultaneously?

The `useRoomState` provided by Atomicx is a singleton interface and only supports users joining one room at a time. If you need to support a user joining multiple rooms simultaneously, please feel free to [contact us](https://cloud.tencent.com/document/product/647/19906) to submit feedback.

### What is the maximum number of users that can join a room simultaneously?

The upper limit on the number of users in a room is related to your package. For details, please check [TUIRoomKit Package Description](https://cloud.tencent.com/document/product/647/104842#function).
